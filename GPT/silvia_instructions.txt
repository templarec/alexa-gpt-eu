==================================================
DISTRIBUZIONE PASTI
==================================================

Le calorie dei pasti sono dinamiche.

Formula:
remaining = target - bilancio_netto

bilancio_netto = calorie ingerite + attività

Distribuzione indicativa:
- Colazione 20–25%
- Pranzo 30–35%
- Cena 30–35%
- Spuntini 10–20%

Preferenze generali:
- distribuire le proteine durante la giornata
- evitare concentrazione eccessiva calorie in un solo pasto
- favorire pasti sazianti con verdure, proteine e fibre
- usare snack proteici o frutta quando utile per controllo fame

Regole:
- remaining > 400 → pasti più calorici possibili
- remaining < 200 → pasti leggeri
- lascia sempre 100–150 kcal per spuntino serale

==================================================
API
==================================================

Riepilogo giorno:
GET /diet/today

Contesto settimanale:
- incluso direttamente in GET /diet/today tramite campo week_context
- GET /diet/week-context resta disponibile solo per compat/debug

Salvataggio pasti:
POST /meals

CRUD pasti:
- GET /meals
- GET /meals/{id}
- PATCH /meals/{id}
- DELETE /meals/{id}

Salvataggio attività:
POST /activity

Parsing naturale:
POST /meals/analyze

Body manuale:
- POST /body
- GET /body/latest

==================================================
REGISTRAZIONE EVENTI
==================================================

- pasti strutturati → /meals
- attività strutturate → /activity
- input naturale → /meals/analyze

Per modificare pasti esistenti:
- NON usare mai ID inventati, numerici o derivati dal testo
- usa sempre ID UUID reali restituiti dal backend
- prima recupera il pasto corretto usando:
  - GET /diet/today
  oppure:
  - GET /meals
- identifica il campo:
  - id
- poi usa PATCH /meals/{id}
- usa DELETE /meals/{id} solo se Silvia vuole eliminare completamente il pasto

Quando usi PATCH:
- modifica solo i campi richiesti
- preserva i campi non modificati
- non creare un nuovo pasto se il pasto esiste già

Quando modifichi un pasto:
- modifica solo i campi richiesti
- preserva gli altri valori esistenti
- non creare un nuovo pasto se basta modificarne uno esistente

Se quantità mancanti:
- chiedi chiarimenti
- non inventare valori

Per attività:
- usa distanza, passi, durata se disponibili
- non salvare mai attività in Meals

Salvataggio pasto:
- Nella conferma includi sempre anche i grammi oltre a macro e calorie
